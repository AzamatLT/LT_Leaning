Производительность дисков — важный аспект в работе систем хранения данных. Вот основные термины, связанные с этой темой:

IOPS (Input/Output Operations Per Second)
Количество операций ввода-вывода, которые диск может выполнить за секунду. Важный показатель для оценки производительности, особенно в задачах с высокой нагрузкой на чтение/запись.

Пропускная способность (Throughput)
Объем данных, который может быть передан через диск за единицу времени (обычно измеряется в МБ/с или ГБ/с). Зависит от скорости интерфейса и возможностей накопителя.

Задержка (Latency)
Время, которое требуется диску для выполнения одной операции ввода-вывода. Включает время поиска (seek time), время вращения (rotational latency для HDD) и время передачи данных.

Время поиска (Seek Time)
Время, необходимое для позиционирования головки чтения/записи над нужной дорожкой на диске (актуально для HDD).

Время вращения (Rotational Latency)
Время, которое требуется диску (HDD), чтобы нужный сектор оказался под головкой чтения/записи. Зависит от скорости вращения диска (например, 5400 RPM, 7200 RPM).

Скорость чтения/записи (Read/Write Speed)
Скорость, с которой данные могут быть считаны или записаны на диск. Обычно измеряется в МБ/с.

Кэш-память (Cache)
Быстрая память на диске, используемая для временного хранения данных и ускорения операций чтения/записи.

RAID (Redundant Array of Independent Disks)
Технология объединения нескольких дисков в массив для повышения производительности, надежности или обоих параметров.

Фрагментация (Fragmentation)
Состояние, при котором файлы на диске (HDD) разбиты на фрагменты и записаны в разных местах, что может снижать производительность.

SSD (Solid State Drive)
Накопитель на основе флеш-памяти, который не имеет движущихся частей и обеспечивает высокую производительность по сравнению с HDD.

HDD (Hard Disk Drive)
Традиционный жесткий диск с магнитными пластинами и механическими головками чтения/записи.

NVMe (Non-Volatile Memory Express)
Протокол для SSD, разработанный для ускорения работы с флеш-памятью через интерфейс PCIe.

SATA (Serial ATA)
Интерфейс для подключения HDD и SSD, ограниченный пропускной способностью (до 600 МБ/с для SATA III).

SAS (Serial Attached SCSI)
Интерфейс, используемый в серверных системах, обеспечивающий более высокую производительность и надежность по сравнению с SATA.

Queue Depth
Глубина очереди запросов, которая показывает, сколько операций ввода-вывода может быть обработано одновременно.

Trim
Команда, используемая в SSD для оптимизации производительности и увеличения срока службы, позволяющая освобождать блоки данных, которые больше не используются.

Wear Leveling
Технология в SSD, которая равномерно распределяет запись данных по ячейкам памяти для увеличения срока службы накопителя.

S.M.A.R.T. (Self-Monitoring, Analysis, and Reporting Technology)
Технология мониторинга состояния диска, которая помогает предсказать возможные сбои.

4K Random Read/Write
Производительность диска при случайном чтении/записи небольших блоков данных (4 КБ), что важно для многих рабочих нагрузок.

Sequential Read/Write
Производительность диска при последовательном чтении/записи больших объемов данных.

Эти термины помогают оценить и сравнить производительность различных типов дисков и выбрать подходящее решение для конкретных задач.

Какие основные файловые системы есть
Файловые системы — это способы организации, хранения и управления данными на носителях информации. Вот основные файловые системы, используемые в современных операционных системах:

Для Windows:
FAT32 (File Allocation Table 32)

Старая файловая система, поддерживаемая большинством устройств.

Ограничения: максимальный размер файла — 4 ГБ, максимальный размер раздела — 8 ТБ.

Используется на флеш-накопителях и внешних дисках для совместимости.

exFAT (Extended File Allocation Table)

Улучшенная версия FAT32, поддерживающая большие файлы и разделы.

Нет ограничений на размер файла, максимальный размер раздела — 128 ПБ.

Часто используется на SD-картах и внешних накопителях.

NTFS (New Technology File System)

Основная файловая система для Windows.

Поддерживает большие файлы (до 16 ЭБ), шифрование, сжатие, квоты и журналирование.

Недостаток: ограниченная поддержка в других ОС (например, macOS и Linux).

Для macOS:
HFS+ (Hierarchical File System Plus)

Использовалась в macOS до 2017 года.

Поддерживает большие файлы и разделы, но устарела.

APFS (Apple File System)

Современная файловая система для macOS, iOS и других устройств Apple.

Оптимизирована для SSD, поддерживает шифрование, снимки (snapshots) и быстрое копирование файлов.

Максимальный размер файла и раздела — 8 ЭБ.

Для Linux:
ext4 (Fourth Extended File System)

Наиболее популярная файловая система в Linux.

Поддерживает большие файлы (до 16 ТБ) и разделы (до 1 ЭБ), журналирование и отложенное выделение пространства.

Надежная и производительная.

ext3 (Third Extended File System)

Предшественник ext4, менее производительный, но до сих пор используется в некоторых системах.

XFS

Высокопроизводительная файловая система, разработанная для больших объемов данных.

Поддерживает файлы и разделы размером до 8 ЭБ.

Часто используется в серверных средах.

Btrfs (B-tree File System)

Современная файловая система с поддержкой снимков, сжатия и RAID.

Разработана как замена ext4, но пока менее стабильна.

ZFS (Zettabyte File System)

Продвинутая файловая система с поддержкой снимков, сжатия, дедупликации и RAID.

Используется в серверных и NAS-системах.

Для кросс-платформенного использования:
exFAT

Поддерживается Windows, macOS и Linux.

Идеальна для внешних накопителей, которые используются на разных ОС.

FAT32

Универсальная, но устаревшая файловая система с ограничениями.

Для сетевых файловых систем:
NFS (Network File System)

Используется для обмена файлами между Unix/Linux-системами по сети.

SMB/CIFS (Server Message Block/Common Internet File System)

Используется для обмена файлами между Windows и другими ОС.

Для специализированных устройств:
F2FS (Flash-Friendly File System)

Оптимизирована для флеш-накопителей (SSD, eMMC, SD-карты).

Используется в Android-устройствах и некоторых Linux-дистрибутивах.

ReFS (Resilient File System)

Разработана Microsoft для серверных систем.

Обеспечивает высокую отказоустойчивость и защиту от повреждения данных.

Для оптических дисков:
ISO 9660

Стандартная файловая система для CD/DVD.

UDF (Universal Disk Format)

Используется для DVD и Blu-ray дисков.

Выбор файловой системы зависит от задач, операционной системы и типа носителя. Например, для внешних накопителей часто выбирают exFAT для совместимости, а для серверов — ZFS или XFS.

В Linux dm и sd — это обозначения, связанные с управлением дисками и устройствами хранения данных

Ключевые различия:
Параметр	sd	 
Тип устройства	Физические диски (SATA, USB, NVMe) 
Имена	sda, sdb, sda1, sdb2 
Использование	Хранение данных	 

Ключевые различия:
Параметр	dm
Тип устройства	 Виртуальные устройства (LVM, RAID)
Имена	 dm-0, dm-1, /dev/mapper/...
Использование	 Управление дисками (LVM, шифрование, RAID)


Где посмотреть информацию о дисках и разделах?
lsblk — показывает информацию о всех блочных устройствах, включая sd и dm.

fdisk -l — список дисков и разделов.

dmsetup ls — список устройств, созданных через Device Mapper.

pvdisplay, vgdisplay, lvdisplay — информация о физических томах, группах и логических томах в LVM.
